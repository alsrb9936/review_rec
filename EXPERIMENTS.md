# CF-Aligned Review Gating Experiments

This project now includes a fast review-aware rating-prediction setup for:

- `CF-only`: user/item embedding dot product with biases, no review input.
- `Whole-review fusion`: projected user/item review embeddings are added to CF embeddings without a gate.
- `CFARG`: scalar gates adapt review contribution from CF/review compatibility. The stabilized version adds LayerNorm, cosine similarity in the gate input, negative gate bias initialization, and a learnable `review_scale` initialized to `0.1`.

All variants use MSE training loss and write RMSE/MAE results to CSV.

## Data and leakage policy

The CFARG loader uses `dataset/<dataset>/bert/user_review_emb.npy` and `item_review_emb.npy`, generated by `utils/bert_pro.py` from **train reviews only**. Validation/test target reviews are not embedded or loaded.

For train samples, `data.exclude_target_review: true` builds leave-one-out train review profiles so the current interaction's own review embedding is removed from both the user and item profile. If an entity has no other train reviews, the profile becomes a zero vector.

Smoke-test leakage-relevant command:

```bash
python run.py --config-name cfarg data.dataset=Amazon_All_Beauty_18 experiment.fast_dev_run=true
```

## Main performance

```bash
bash scripts/run_main.sh
```

Single seed/debug run:

```bash
SEEDS="42" FAST_DEV=true bash scripts/run_main.sh
```

Output:

```text
results/cfarg_fixed/main_results.csv
results/cfarg_fixed/gate_stats.csv
```

Columns: `dataset, model, seed, rmse, mae, best_epoch, avg_user_gate, avg_item_gate, review_scale, config_path, checkpoint_path`.

Fixed CFARG main script:

```bash
bash scripts/run_main_cfarg_fixed.sh
```

## Ablation study

```bash
bash scripts/run_ablation.sh
```

Variants: `CF-only`, `Review-only`, `CF + projected review without gate`, `CFARG`.

The previous `projection` alias had the same implementation as `fusion`; it is no longer used by the ablation script.

Output:

```text
results/cfarg_fixed/ablation_results.csv
```

## Noise robustness

```bash
bash scripts/run_noise.sh
```

Supported random replacement ratios: `0.0`, `0.1`, `0.2`, `0.3`.

Side-specific noise:

```bash
NOISE_SIDE=user bash scripts/run_noise.sh
NOISE_SIDE=item bash scripts/run_noise.sh
```

Output:

```text
results/cfarg_fixed/noise_results.csv
```

Columns: `dataset, model, seed, noise_type, noise_ratio, rmse, mae, avg_user_gate, avg_item_gate`.

## Direct Hydra commands

```bash
python run.py --config-name cf_only data.dataset=Amazon_All_Beauty_18 experiment.seed=42
python run.py --config-name whole_review_fusion data.dataset=Amazon_All_Beauty_18 experiment.seed=42
python run.py --config-name cfarg data.dataset=Amazon_All_Beauty_18 experiment.seed=42
python run.py --config-name cfarg_noise_10 data.dataset=Amazon_All_Beauty_18 experiment.seed=42
```

CUDA is auto-detected in `run.py`; if CUDA is unavailable the run falls back to CPU.
